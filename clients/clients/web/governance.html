Governance & security summary

Human-governed autonomy, auditable execution, portable boundaries

Heiwa V1 is designed as an enterprise platform foundation: daemon-first execution, PR-only governance, policy-gated outbound network calls, and local-vs-runtime separation.

Change Control

  • Agents propose changes and open PRs.
  • Humans review, merge, and deploy.
  • CI gates enforce schema, safety, and policy checks.

Secrets & Data

  • No runtime secrets committed to repos.
  • Local secrets stored outside repo and referenced by symlink.
  • Cloud secrets managed in provider variables (Railway/Cloudflare/GitHub).

Execution Boundaries

  • Railway runs control-plane services.
  • Privileged operations run on policy-gated local workers.
  • Public web status is read-only and non-privileged.

V1 provider stack

Railway + Cloudflare + GitHub

Heiwa V1 standardizes on a focused provider stack to reduce operational sprawl. Future platform expansion can add providers only after the core control-plane and worker mesh are stable and verified.
